# Optimism Retro Funding — Strategy Note (do not submit yet)

Retro Funding rewards *demonstrated impact*, not plans. Sentinel is not ready to
apply: there is working code but no adoption evidence yet. Applying early wastes
the shot — badge-holders score on measurable usage.

## When to apply

After M4 ships and there is real usage: the attack-blocked demo is public, v0.1
is on npm with nonzero downloads, and at least one agent framework or wallet SDK
integration is live (ideally on OP Stack chains — Base counts).

## Evidence to collect starting now

Impact claims must be verifiable later. Keep a running log (dated, with links):

- [ ] npm download stats over time (`sentinel-firewall`)
- [ ] GitHub stars/forks/dependents; external PRs and issues
- [ ] Named integrations: which frameworks/SDKs, links to their code using Sentinel
- [ ] Blocked-transaction demos and any real-world saves users report (screenshots,
      tx hashes, write-ups)
- [ ] OP Stack relevance: usage on Base or other OP chains specifically
- [ ] Talks, posts, or third-party coverage mentioning Sentinel

## Framing when the time comes

- Category: developer tooling / security public good
- Impact statement: open, neutral enforcement layer for agent wallets on OP Stack
  chains; the alternative is platform-locked proprietary screening
- Lean on the no-token, no-hosted-dependency, MIT positioning — credible
  neutrality is exactly what retro rounds reward
